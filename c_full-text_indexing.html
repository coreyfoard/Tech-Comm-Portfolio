PDF Full-Text Indexing

Full-text PDF indexing allows embedded text within PDFs to be searched with Quick Search (via the “Everything” option) and Advanced Search (via “Attachment Content”).

You can control how much text in a PDF is indexed in the Search pane of Zotero preferences (default: 500000 characters, 100 pages). Indexing happens automatically in the background when Zotero is idle. You can remove indexed text with the “Clear Index…” button or re-create the index from scratch using the “Rebuild Index…” button. You can check the index status of any PDF attachment by selecting the attachment item in the Zotero library and looking at the “Indexed:” field in the right pane.

If an item isn't being indexed (e.g., if it is not showing up not showing up an 'Everything' Quick Search), verify that the item has searchable text and that the text is properly stored in the PDF (e.g., try to copy text out of the document and ensure that it is high quality). If the PDF has valid text, rebuild the item's index by right-clicking on it and choosing “Reindex Item”. If you are still having issues, please ask a question on the Zotero forums.

  • At this time, only PDF full text content (and plain text files) can be indexed by Zotero. Other document types (e.g., .docx, .odt, .epub) cannot be indexed by Zotero.
